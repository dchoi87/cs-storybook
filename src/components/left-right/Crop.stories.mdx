## 2. Auto Crop

If image is larger than copy, there will be little to no crop. However, if copy is larger, image will crop based off copy height while maintaining its aspect ratio.

Ideal solution for both scenarios but some caveats:

* Using `object-fit: cover` meaning **no support for IE11 and below**
* For mobile, requires fixed height for media iframes due to it becoming a column view
* **Component heights will always vary** due to dependencies on both image and copy

## 3. Fixed Crop

Cropping based off a fixed height. Only relevant for scenarios like the homepage full width. Like the No Crop option, will require a limit on the copy content.

In this example, crop height is fixed at three separate breakpoints:

* Desktop: `280px`
* Tablet: `350px`
* Mobile: `250px`
